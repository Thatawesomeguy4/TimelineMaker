# Timeline Authoring Tool

A browser-based timeline authoring application for creating, editing, importing, exporting, and collaboratively maintaining structured timelines.

This project is designed for use cases such as incident response timelines, investigation timelines, event reconstruction, outage reviews, and operational chronology tracking.

## Features

### Current Features

- Web-based timeline editor
- Start menu with options to:
  - Create a new timeline
  - Open/import a local timeline file
  - Join a collaboration room
- Timeline event creation, editing, duplication, and deletion
- Event customization options:
  - Title
  - Description / notes
  - Start and end time
  - Lane
  - Category
  - Event type
  - Status
  - Severity
  - Confidence
  - Icon
  - Color
  - Display style
  - Actor / owner
  - Location
  - Source
  - Evidence URL
  - Tags
- Event editing through a popup modal
- Card-based timeline view
- Horizontal scrolling timeline diagram view
- Sort modes:
  - Sort by time
  - Manual order
- Drag-and-drop event reordering in manual order mode
- Local timeline save/export to `.timeline.json`
- Local timeline import from `.timeline.json`
- Import validation using Zod
- Collaboration room selection
- Real-time event synchronization using Yjs and y-websocket
- Docker-based development environment
- PostgreSQL and Redis containers included for future API persistence/scaling work

Example:

```json

  "id": "incident-2026-001",
  "title": "Incident Timeline",
  "description": "Timeline of investigation events",
  "timezone": "America/New_York",
  "lanes": [
    {
      "id": "security-ops",
      "name": "Security Operations"
    }
  ],
  "events": [
    {
      "id": "evt-001",
      "title": "Alert received",
      "description": "Initial alert generated by monitoring system.",
      "start": "2026-06-18T08:00:00-04:00",
      "end": null,
      "order": 0,
      "laneId": "security-ops",
      "category": "Detection",
      "eventType": "Alert",
      "status": "Completed",
      "severity": "High",
      "color": "#2563eb",
      "icon": "Alert",
      "displayStyle": "Solid",
      "tags": ["security", "alert"],
      "source": "Manual entry",
      "evidenceUrl": "",
      "confidence": "High",
      "actor": "Security Operations",
      "location": "SOC",
      "createdAt": "2026-06-18T08:00:00-04:00",
      "updatedAt": "2026-06-18T08:00:00-04:00"
    }
  ],
  "createdAt": "2026-06-18T08:00:00-04:00",
  "updatedAt": "2026-06-18T08:00:00-04:00"
}
```

## Getting Started

### Prerequisites

Install the following on your workstation:

- Docker Desktop or Docker Engine
- Docker Compose
- Git
- Visual Studio Code
- A modern browser such as Microsoft Edge or Chrome

Node.js does not need to be installed directly on the host machine. Node.js runs inside Docker containers.

## Windows Development Notes

For best performance on Windows, especially with Docker Desktop and WSL 2, it is recommended to store the project inside the WSL filesystem.

Recommended location:

```text
\\wsl$\Ubuntu\home\<your-user>\projects\timeline-authoring-tool
```

Alternative Windows path:

```text
C:\Dev\timeline-authoring-tool
```

If using a Windows path, file watching may require polling, which is already enabled in the Vite configuration and Docker Compose environment.

## Example Docker Compose Yjs Service

The Yjs service should use `@y/websocket-server`.

```yaml
yjs:
  image: node:24-bookworm
  container_name: timeline-yjs
  working_dir: /workspace/apps/web
  volumes:
    - ./:/workspace
    - web_node_modules:/workspace/apps/web/node_modules
  ports:
    - "1234:1234"
  environment:
    HOST: 0.0.0.0
    PORT: 1234
  command: sh -c "./node_modules/.bin/y-websocket"
  depends_on:
    - web
```

Install the server dependency in the web app container:

```bash
docker compose exec web npm install @y/websocket-server
```

## Using the Application

### Start Menu

When the app loads, users are presented with a start menu.

Available options:

- New Timeline
- Open Timeline
- Collaborate

### New Timeline

Creates a blank local timeline workspace.

### Open Timeline

Opens a local file picker and imports a `.timeline.json` file.

The imported file is validated using the timeline schema before being loaded.

### Collaborate

Allows the user to enter a collaboration room ID.

Users who enter the same room ID connect to the same Yjs room and can see timeline changes in real time.

Example room ID:

```text
incident-2026-test
```

## Views

### Card View

The card view displays each event as an editable event card.

Users can:

- Edit event title directly
- Open the full event editor
- Duplicate events
- Delete events
- Drag and drop events in manual order mode

### Horizontal Timeline View

The horizontal view displays events on a scrolling timeline diagram.

Events alternate above and below the center line. Clicking an event card or event node opens the event editor.

## Sorting Modes

### Sort by Time

Events are ordered by the event `start` field.

Drag-and-drop is disabled in this mode to prevent accidental modification of timeline presentation order.

### Manual Order

Events are ordered by the event `order` field.

Drag-and-drop is enabled in this mode. Reordering events does not change evidence timestamps.

## Local Save and Import

### Save

The Save button exports the current timeline as a `.timeline.json` file.

Where supported, the browser File System Access API can prompt for a save location. Otherwise, the browser downloads the file normally.

### Import

The Import button opens a local file picker and loads a timeline file.

Imported timelines are validated before loading.

## Collaboration

Collaboration is powered by:

```text
Yjs
y-websocket
@y/websocket-server
```

The collaboration flow is:

```text
User joins room ID
TimelineEditor calls useTimelineCollaboration(roomId)
Client connects to Yjs websocket server
Events and metadata are synchronized through shared Yjs maps
Users in the same room see timeline changes
```

The current collaboration model syncs:

```text
timeline metadata
lanes
events
event order
event edits
event deletion
event duplication
```

Current limitations:

- Last update wins for concurrent edits to the same event
- User presence is set in the Yjs provider but not yet displayed in the UI
- Collaboration state is in-memory unless persistence is added to the Yjs backend
- Authentication and authorization are not yet enforced on collaboration rooms

## Troubleshooting

### Yjs container fails with npm 404

Incorrect package:

```text
@y/y-websocket-server
```

Correct package:

```text
@y/websocket-server
```

Install:

```bash
docker compose exec web npm install @y/websocket-server
```

Then restart:

```bash
docker compose restart yjs
```

### Vite cannot resolve `zod`

If `packages/timeline-schema/src/validation.ts` imports `zod`, install `zod` where the shared package can resolve it.

Recommended root install:

```bash
docker compose exec web bash -lc "cd /workspace && npm install zod"
```

Then clear Vite cache:

```bash
docker compose exec web bash -lc "rm -rf /workspace/apps/web/node_modules/.vite"
docker compose restart web
```

### Tailwind command `npx tailwindcss init -p` fails

Tailwind CSS v4 no longer uses the old v3 initialization flow.

Use the Vite plugin approach:

```bash
npm install tailwindcss @tailwindcss/vite
```

In `vite.config.ts`:

```ts
import tailwindcss from "@tailwindcss/vite";
```

Add the plugin:

```ts
plugins: [react(), tailwindcss()]
```

In `src/index.css`:

```css
@import "tailwindcss";
```

### TypeScript complains about explicit file extensions

For the frontend Vite app, use bundler module resolution.

Example:

```json
{
  "compilerOptions": {
    "moduleResolution": "bundler"
  }
}
```

### `erasableSyntaxOnly` errors

If the frontend TypeScript config has this:

```json
"erasableSyntaxOnly": true
```

and it causes issues during development, change it to:

```json
"erasableSyntaxOnly": false
```

### React type namespace errors

If using event types like:

```ts
React.ChangeEvent<HTMLInputElement>
```

either import React as a namespace or use a type-only import:

```ts
import type { ChangeEvent } from "react";
```

Then:

```ts
function handleChange(event: ChangeEvent<HTMLInputElement>) {
  // ...
}
```
